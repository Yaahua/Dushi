# 《都市余烬》通用技术架构与内容交付清单

## 一、 技术架构选型 (通用底层)

*   **后端**：Python (FastAPI) / Node.js (Express) - 采用微服务/模块化设计。
*   **数据库**：PostgreSQL - 存储所有动态配置与玩家数据。
*   **缓存/队列**：Redis - 处理实时 Tick 与异步消息。
*   **前端**：Web-based Terminal (React + WebSocket) - 确保跨平台一致性。

## 二、 内容交付清单 (开放式列表)

请制作人根据以下分类，源源不断地提供规划内容。AI 将按需将其转化为代码：

### 1. 城市规则与数值 (Rules & Stats)
*   [ ] 基础属性定义 (体力、饥饿、健康等上限与消耗公式)。
*   [ ] 经济数值 (物价、工资、税率、手续费、交易限额)。
*   [ ] 时间比例 (RT 与 GT 的具体换算数值)。

### 2. 地理与设施 (Geography & Infrastructure)
*   [ ] 区域描述文本 (各区的环境氛围、ASCII 效果要求)。
*   [ ] 交通参数 (各工具的费用、耗时、风险、信号限制)。
*   [ ] 建筑/POI 列表 (各地点功能、准入门槛)。

### 3. 社会、职业与政策 (Society & Policy)
*   [ ] 职业系统规划 (各职业的准入标签、薪资、排班要求)。
*   [ ] education 系统规划 (课程内容、考试题库、证书效力)。
*   [ ] 法律政策 (违规定义、惩罚时长、执法逻辑)。
*   [ ] 城市动态政策 (如：天气影响逻辑、季节性规则)。

### 4. 角色、NPC 与社交 (Characters & Social)
*   [ ] 角色背景与属性分配规则。
*   [ ] NPC 档案 (人设、对话文本、行为轨迹、记忆门槛)。
*   [ ] 社交动作逻辑 (交换号码、打听、赠送等文本与概率)。

### 5. 交互工具与系统 (Tools & Systems)
*   [ ] 手机 App 功能描述 (地图、简讯、叫车、客软件的具体交互)。
*   [ ] 实物交互逻辑 (便条、物品传递、环境痕迹文本)。
*   [ ] 犯罪/冲突逻辑 (预备链要求、成功率公式、后果描述)。

---

## 三、 交付说明
制作人可按需、按模块、不限顺序地提供上述任何内容。AI 负责根据交付内容进行技术实现，并确保系统间的关联性。
