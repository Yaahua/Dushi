# NPC 系统设定框架

## 系统概述

NPC (Non-Player Character) 是城市生态的重要组成部分，包括固定NPC（店员、公务员等）和动态NPC（路人、流浪者等）。

## NPC 分类

### 按功能分类

| 类型 | 说明 | 示例 |
| :--- | :--- | :--- |
| `service` | 服务提供者，提供固定功能交互 | 便利店店员、房东、银行职员 |
| `story` | 剧情NPC，承载叙事功能 | 关键任务角色、信息来源 |
| `ambient` | 环境NPC，营造氛围 | 路人、流浪者、背景角色 |
| `faction` | 派系NPC，代表特定群体利益 | 帮派成员、工会代表 |

### 按移动性分类

| 类型 | 说明 |
| :--- | :--- |
| `static` | 固定在特定位置，不移动 |
| `patrol` | 在指定区域内巡逻 |
| `migrant` | 在城市中自由移动 |
| `scheduled` | 按时间表在不同地点出现 |

## 文件组织

```
npc/
├── README.md              # 本文件
├── _template.yaml         # NPC配置模板（复制此文件创建新NPC）
├── _common_dialogues.yaml # 通用对话库
├── 世界公寓/              # 区域文件夹
├── 主城区/
└── 动态生成/              # 动态NPC生成规则
```

## 关联系统

- **地图系统**: NPC 所在位置由地图坐标定义
- **事件系统**: NPC 可触发或参与事件
- **时间系统**: NPC 行为受时间影响
- **关系系统**: NPC 与玩家的关系影响交互

## 数据格式规范

详见 `_template.yaml` 模板文件。
