# 代码问题分析与重构方案

## 为何这次搞这么久

### 1. 根本原因：代码混乱

**问题清单**：

| 问题 | 影响 |
|------|------|
| 过期逻辑分散在多个地方 | 难以追踪哪个操作过期了什么 |
| `expireNarrativeLinks` 和 `expireActionCards` 职责不清 | 容易漏掉某些过期场景 |
| `links` 可以来自 `actions` 或 `onActionComplete` | 来源不统一，管理混乱 |
| 隐式依赖：场景切换时应该过期哪些内容？ | 靠记忆而非显式规则 |

### 2. 我的问题

- **没有先理解全貌**就动手修复
- **过度设计**：一开始就想做通用系统
- **反复修补**：发现问题就加逻辑，没有重构

## 重构方案：让代码易于 Debug

### 目标
1. 所有过期逻辑**集中管理**
2. 每个操作的副作用**显式声明**
3. 添加**调试日志**便于追踪
4. **类型安全**：不允许隐式操作

### 方案：操作流水线模式

```typescript
// 每个操作定义为一个流水线步骤
interface OperationStep {
  type: 'buy' | 'eat' | 'goto' | 'interact';
  execute: (state: GameState) => OperationResult;
}

interface OperationResult {
  state: GameState;           // 新状态
  expiredTypes: string[];     // 要过期的卡片类型
  expireSceneLinks: boolean;  // 是否过期场景链接
  log: string[];              // 调试日志
}

// 统一的执行器
function executeOperation(
  state: GameState, 
  step: OperationStep
): { state: GameState; debug: string[] } {
  const result = step.execute(state);
  
  // 集中处理过期逻辑
  let narrative = result.state.narrative;
  
  // 1. 过期指定类型的卡片
  for (const type of result.expiredTypes) {
    narrative = expireActionCards(narrative, type);
  }
  
  // 2. 过期场景链接（如果需要）
  if (result.expireSceneLinks) {
    narrative = expireNarrativeLinks(narrative);
  }
  
  return {
    state: { ...result.state, narrative },
    debug: [
      `[操作] ${step.type}`,
      `[过期类型] ${result.expiredTypes.join(', ') || '无'}`,
      `[过期场景链接] ${result.expireSceneLinks}`,
      ...result.log,
    ],
  };
}
```

### 使用示例

```typescript
case 'BUY_ITEM': {
  const result = executeOperation(state, {
    type: 'buy',
    execute: (s) => {
      // ... 购买逻辑 ...
      return {
        state: newState,
        expiredTypes: ['buy'],        // 过期之前的购买卡片
        expireSceneLinks: true,       // 过期原始场景链接
        log: ['购买矿泉水成功'],
      };
    },
  });
  
  // 调试：可以在控制台看到这些日志
  console.log(result.debug.join('\n'));
  return result.state;
}
```

### 输出示例

```
[操作] buy
[过期类型] buy
[过期场景链接] true
购买矿泉水成功
```

## 另一个方案：配置驱动（更简单）

如果重构成本太高，可以用配置表管理：

```typescript
// 每个操作的行为配置
const OPERATION_CONFIG: Record<string, {
  expireTypes: string[];
  expireSceneLinks: boolean;
  showActionCard: boolean;
}> = {
  buy: {
    expireTypes: ['buy'],
    expireSceneLinks: true,
    showActionCard: true,
  },
  eat: {
    expireTypes: ['eat'],
    expireSceneLinks: true,
    showActionCard: true,
  },
  goto: {
    expireTypes: ['buy', 'eat', 'interact'],  // 移动时过期所有操作卡片
    expireSceneLinks: true,
    showActionCard: false,
  },
};

// reducer 中统一使用
function applyOperation(
  state: GameState,
  operationType: string,
  updateFn: (s: GameState) => GameState
): GameState {
  const config = OPERATION_CONFIG[operationType];
  if (!config) return updateFn(state);
  
  let newState = updateFn(state);
  let narrative = newState.narrative;
  
  // 过期指定类型的卡片
  for (const type of config.expireTypes) {
    narrative = expireActionCards(narrative, type);
  }
  
  // 过期场景链接
  if (config.expireSceneLinks) {
    narrative = expireNarrativeLinks(narrative);
  }
  
  return { ...newState, narrative };
}
```
