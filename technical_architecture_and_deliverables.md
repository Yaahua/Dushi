# 《都市余烬》开发技术架构与交付清单

## 引言

本文件旨在明确《都市余烬》项目的技术架构选型，并详细列出您需要向我提供的具体交付物。

## 一、技术架构建议

### 1. 后端服务 (Backend Service)
*   **语言与框架**：**Python (FastAPI)** 或 **Node.js (Express.js)**。
*   **数据库**：**PostgreSQL**。
*   **异步任务队列**：**Redis** 搭配 **Celery** 或 **BullMQ**，处理 City Tick 与离线结算。

### 2. 游戏界面 (Game Interface)
*   **界面类型**：**命令行界面 (CLI)** 或 **基于 Web 的终端模拟器**。
*   **前端技术**：若采用 Web 终端，使用 **React** 搭配 **WebSocket**。

### 3. 部署与运营
*   **容器化**：**Docker**。
*   **云平台**：**GCP** 或 **AWS**。

## 二、您的交付清单

### 阶段一：核心系统基础搭建
1.  **City Tick 参数**：时钟比例（RT 1天 = GT 7天）、离线策略触发规则。
2.  **界面设计稿 (文字版)**：状态栏、叙事流、指令输入区及手机/背包等 ASCII 布局。
3.  **数据库需求**：所有需要存储的数据字段（玩家、物品、NPC 等）。

### 阶段二：基础生存与交互
1.  **角色创建配置**：四维分配规则、年龄影响、饥饿/健康数值定义。
2.  **基础指令与场景文本**：移动指令列表、交互指令列表、场景初始描述。
3.  **经济系统数值**：初始资金、账期、房租水电、交易限额。
4.  **初始难民教学文本**：教学内容、测试题库、期间简报范例。

### 階段三：异步社交与手机系统
1.  **手机功能文本**：地图、电话簿、简讯（延迟机制）、叫车、客软件、紧急呼叫的具体文本。
2.  **实物社交文本**：便条范例、物品传递逻辑、环境改造效果。
3.  **离线策略文本**：各策略详细描述、随机事件池。

### 阶段四：进阶系统与内容扩展
1.  **教育系统文本**：职教/大学课程、门槛、考试题库、认证标签效力。
2.  **高门槛犯罪文本**：工具、侦察、三要素判定文本、成功率修正、惩罚后果。
3.  **城市基础设施文本**：设施层级描述、接管条件、交通/着装功能。
4.  **NPC 行为与文本**：人设、对话分支、记忆点、关系反应逻辑。
