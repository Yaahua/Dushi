# 《都市餘燼》開發技術架構與交付清單

## 引言

本文件旨在明確《都市餘燼》專案的技術架構選型，並詳細列出您（作為遊戲製作人與內容創作者）需要向我（Manus AI，技術實作方）提供的具體交付物。清晰的技術選型將確保開發效率與系統穩定性，而明確的交付清單則能優化協作流程，加速專案進度。

## 一、技術架構建議

基於《都市餘燼》「純文字、單窗口、異步多人」的核心設計理念，我們將採用以下技術棧來構建遊戲：

### 1. 後端服務 (Backend Service)

*   **語言與框架**：**Python** 搭配 **FastAPI** 或 **Node.js** 搭配 **Express.js**。
    *   **考量**：Python 在數據處理、異步任務管理方面有成熟的生態，適合處理遊戲的複雜邏輯和定時任務（City Tick）。Node.js 則在異步 I/O 方面表現出色，適合處理大量併發請求。兩者皆能良好支援遊戲的異步特性。
*   **資料庫**：**PostgreSQL** 或 **TiDB**。
    *   **考量**：PostgreSQL 是一款功能強大、穩定可靠的開源關係型資料庫，適合儲存玩家數據、城市狀態、物品記憶 ID 等結構化數據。TiDB 則是一個雲原生的分散式 SQL 資料庫，若未來考慮遊戲規模擴展至極大規模，能提供更好的水平擴展能力。
*   **異步任務隊列**：**Redis** 搭配 **Celery (Python)** 或 **BullMQ (Node.js)**。
    *   **考量**：用於處理「City Tick」的定時任務、玩家離線策略的異步結算、延遲簡訊發送等非即時性操作，確保系統響應速度與穩定性。

### 2. 遊戲介面 (Game Interface)

*   **介面類型**：**命令行介面 (CLI)** 或 **基於 Web 的終端模擬器**。
    *   **考量**：完全符合「純文字終端美學」的設計。若採用 CLI，可直接透過 SSH 或 WebSocket 連接。若採用 Web 終端模擬器，則可提供更廣泛的跨平台支援，並能更靈活地控制文字渲染與交互效果。
*   **前端技術 (若採用 Web 終端)**：**React** 或 **Vue** 搭配 **WebSocket**。
    *   **考量**：用於構建響應式、高效能的 Web 終端介面，透過 WebSocket 實現與後端服務的即時雙向通訊，確保指令輸入與敘事流的流暢體驗。

### 3. 部署與營運 (Deployment & Operations)

*   **容器化**：**Docker**。
    *   **考量**：將遊戲的各個服務（後端、資料庫、任務隊列）容器化，便於部署、管理與擴展。
*   **雲平台**：**Google Cloud Platform (GCP)** 或 **AWS**。
    *   **考量**：提供穩定可靠的基礎設施服務，如虛擬機、資料庫服務、負載均衡等，支援遊戲的全球部署與擴展。

## 二、您的交付清單 (按開發階段)

以下是根據《都市餘燼》開發規劃，您需要向我提供的具體內容清單。請以清晰、結構化的文字檔案（例如 Markdown 或純文本）形式交付，並確保內容的完整性與一致性。

### 階段一：核心系統基礎搭建

1.  **City Tick 系統參數**：
    *   「世界時鐘」與「個人時鐘」的具體時間流逝比例（例如：RT 1天 = GT 7天）。
    *   離線策略的觸發條件與規則。
2.  **單窗口介面設計稿 (文字版)**：
    *   頂部狀態欄的文字元素（例如：`[≡] 都市余烬 周四/晴 ¥2,450 📱 💌3 [👤]`）及其動態更新邏輯。
    *   中央敘事流的文字排版範例與滾動行為描述。
    *   底部指令輸入區的文字提示與「[发送] [快捷▼]」等按鈕的文字表示。
    *   所有內聯展開的臨時區塊（如手機屏幕、背包）的 ASCII 藝術設計與文字內容範例。
3.  **資料庫需求清單**：
    *   所有需要儲存的數據類型及其主要欄位（例如：玩家 ID、角色名稱、四維屬性、金錢、物品清單、城市位置、NPC ID、NPC 狀態、簡訊內容等）。
    *   數據之間的關係（例如：玩家擁有物品、NPC 位於某個地點）。

### 階段二：基礎生存與交互

1.  **角色創建配置**：
    *   100 點分配至四維（體質/神經質/外向/智力）的具體規則與單項上限（40點）。
    *   青年/中年/老年年齡段的具體定義，以及對遊戲的影響（例如：青年免疫殺害，老年死亡風險遞增）。
    *   基礎狀態（飢餓、健康）的數值定義、變化速率、觸發條件與影響（例如：飢餓值低於 X 觸發「飢餓」狀態，影響體力恢復）。
2.  **基礎指令與場景文本**：
    *   所有基礎移動指令（例如：`去便利店`、`去公園`）的列表。
    *   其他核心交互指令（例如：`應聘`、`打開 手机`、`申請入學`）的列表。
    *   各個場景的初始描述文本（例如：「你醒來在公寓裡...」）。
3.  **經濟系統數值**：
    *   初始資金：500 元現金。
    *   發薪日：每月 15 日 (GT)。
    *   帳單日：每月 1 日 (GT)。
    *   房租、水電、話費的具體數值（例如：¥50/月）。
    *   ATM 取現手續費比例（5%），轉帳延遲時間（GT 1天）。
    *   交易限制：單筆現金限額（2,000元）、日累積限額（5,000元）、新帳號保護期內每日現金限額（500元）。
4.  **初始難民教學文本**：
    *   「難民服務中心」的強制安全簡報內容（基礎生存知識，防騙/帳單/緊急呼叫）。
    *   入境時 7 題測試的題目與正確答案。
    *   「期間簡報」的範例文本，特別是離線 7 天後的聚合簡報內容。

### 階段三：異步社交與手機系統

1.  **手機功能文本與邏輯**：
    *   **地圖**：文字描述如何呈現「實時定位 + 周邊玩家密度（模糊）」，以及「無導航路徑」的文字提示。
    *   **電話簿**：`交換號碼` 指令的文本，上限 50 個號碼的提示，以及可填假名的文本說明。
    *   **簡訊**：延遲時間（GT 30分鐘-2小時）、70 字限制、夜間順延至 GT 08:00 的文本提示，以及無群聊的說明。提供簡訊對話範例。
    *   **叫車**：發布需求、司機接單後顯示 ETA 的文本，深夜司機可查看乘客背包價值（犯罪誘因）的文本描述，以及必須線下碰頭的文本提示。
    *   **客軟體**：純信息板的文本描述，招聘/租房/二手信息的範例內容，無聊天功能、無在線支付的說明，所有交易線下完成的提示，以及評價延遲（GT 3天）的說明。
    *   **緊急呼叫**：110/120/119 的文本說明，一鍵發送定位的提示，假報警懲罰（GT 7天通訊封禁）的文本，120 收費（GT 200元，無錢拒載）的文本。
    *   手機使用限制的文本描述（可用/禁用狀態、信號限制）。
2.  **實物社交文本**：
    *   便條系統的文本範例（貼在物體表面、可回復/撕掉/無視）。
    *   物品傳遞的文本範例（例如：「借你傘，記得還」），以及物品帶記憶 ID 可追溯歷任主人的文本說明。
    *   環境改造（刻字、塗鴉）的文本範例與持久化效果描述。
3.  **離線策略文本**：
    *   玩家下線時可選擇的策略列表（去學校/去上班/在家resting/外出求職/黑市蹲點）的詳細文本描述。
    *   每種策略在離線期間可能觸發的隨機事件文本，以及資源耗盡時自動降級為「在家resting」的文本提示。

### 階段四：進階系統與內容擴展

1.  **教育系統文本與數值**：
    *   **基礎生存教學**：難民服務中心強制安全簡報的詳細內容。
    *   **職業教育**：各區職業培訓中心的課程名稱、短期課程時間（GT 7-14天）、項目制考核的文本描述，以及產出（基礎職業許可：店員、司機、日結工等）的文本。
    *   **大學教育**：大學區的文本描述，准入門檻（居住證 GT 30天以上、學費 GT 2,000-5,000元/學期、入學考試內容）的文本。學期制（GT 30天=現實約5天）、可中途退學（學費不退）、離線策略新增「去大學」（深度學習模式，需教材）的文本說明。官方課程與社區課程的內容範疇，以及產出（高階認證標籤、解鎖高薪職業、知識特權）的文本。
2.  **高門檻犯罪文本與數值**：
    *   **預備階段**：工具（黑市購買凶器價格範圍、占背包格、徒手成功率0%）的文本。偵查（`跟踪` 連續 3 次以上、黑市通話記錄竊取價格）的文本。三要素（地點、時間、狀態）的具體文本描述。
    *   **成功率公式**：公式中各項加成/減益的文本描述（例如：`+10%（有武器）`、`-20%（目標有防備）`）。
    *   **後果**：失敗逃脫（通緝、RT 24小時封禁）的文本。成功（「殺人者」標籤、RT 7天硬封禁、GT 30天冷卻期）的文本。
3.  **城市基礎設施文本**：
    *   中層與頂層基礎設施的詳細文本描述，包括控制方、內容，以及玩家持股 51% 可接管的文本說明。
    *   交通（步行、地鐵、打車）的文本描述與數值（例如：步行 GT 3分鐘=RT 15秒，消耗體力；地鐵每 GT 10分鐘一班）。
    *   著裝（保暖度、整潔度、隱蔽度）的功能性文本描述。
4.  **NPC 行為與對話文本**：
    *   所有關鍵 NPC 的詳細人設、對話文本、行為模式與記憶點。
    *   NPC 之間的關係網與對玩家行為的反應邏輯（例如：玩家犯罪後，某些 NPC 可能會改變態度）。

### 階段五：測試、優化與部署

1.  **測試反饋**：
    *   遊戲數值平衡的具體調整建議。
    *   玩法體驗的優化建議。
    *   所有文本內容的校對與潤飾。
2.  **社群測試規劃**：
    *   社群測試的具體目標與測試重點。
    *   期望收集的玩家反饋類型。

## 三、交付方式與協作流程

*   **交付格式**：請以清晰、結構化的 Markdown (`.md`) 或純文本 (`.txt`) 檔案形式交付所有內容。對於複雜的數值表格或流程圖，可以使用 Markdown 表格或文字描述。
*   **版本控制**：建議您使用 GitHub 或其他版本控制系統來管理您的文本內容，並將儲存庫連結提供給我，以便我能隨時獲取最新版本。
*   **溝通頻率**：在每個階段開始前，我們將確認該階段的交付清單。您完成交付後，我將進行技術實作，並定期向您匯報進度。
