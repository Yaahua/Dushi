# 《都市余烬》开发路径与技术栈建议文档

## 项目背景与核心架构概述

《都市余烬》被定义为一款纯文字、单窗口、异步多人的现代都市生活模拟器。全服玩家将共享同一物理城市，作为新移民从零开始生存。根据《未细化暂定大纲》与《开发规划》，游戏采用了**“动态数据驱动 (Data-Driven)”**的模块化架构，制作人负责规划城市的政策、文本与数值，而工程端则负责构建通用的技术框架。

项目的核心系统包含八个维度：时间系统（双层时钟与City Tick）、出身与注册、经济系统（双资产结构与账期）、生命与死亡、身份与职业、城市基础设施、叙事与事件、以及基于终端美学的交互界面。

## 技术栈选型建议

根据《技术架构与交付清单》中的初步规划，并结合现代文字 MMO 的最佳实践，我为您整理了以下技术栈选型建议。请您审查并确认是否采用：

| 层级 | 推荐技术选型 | 备选方案 | 选择理由与优势 |
| :--- | :--- | :--- | :--- |
| **后端框架** | **Python (FastAPI)** | Node.js (Express/NestJS) | FastAPI 原生支持异步 (Asyncio) 和 WebSocket，非常适合处理游戏中的高并发 Tick 运算和实时状态推送。同时，Python 在处理复杂逻辑（如动态公式解析、AI 接入）时具有显著优势。 |
| **数据库** | **PostgreSQL** | MySQL | PostgreSQL 在处理复杂的 JSON 数据结构（用于存储动态配置和玩家状态）和空间数据（未来可能的地图坐标计算）方面表现优异，且具备强大的事务处理能力，适合经济系统的双资产管理。 |
| **缓存与消息队列** | **Redis** | RabbitMQ | Redis 不仅可以作为高速缓存，其 Pub/Sub 机制完美契合全服状态同步和异步消息队列的需求，是处理实时 Tick 结算的核心组件。 |
| **前端框架** | **React + WebSocket** | Vue.js | React 拥有丰富的生态，适合构建模块化的终端 UI 组件（如 Header、MainStream、InputArea）。结合 WebSocket，可以实现打字机效果、状态闪烁等实时交互反馈。 |
| **部署架构** | **Docker + Nginx** | 裸机部署 | 容器化部署便于环境隔离和快速扩展，符合“静态验证(MVP) → 全服同城(分区服务器)”的渐进式部署路径。 |

**需要您的意见 (节点 1)：**
请问您对上述基于 Python (FastAPI) + React 的技术栈选型是否满意？如果您更倾向于 Node.js 或有其他偏好，请告诉我，我将调整后续规划。

## 渐进式开发路径规划

为了确保项目稳步推进，遵循“制作人提供数据，AI 负责实现”的协作流程，我将开发路径划分为以下四个主要阶段。每个阶段都包含具体的开发任务和需要您审查的交付物。

### 第一阶段：基础设施搭建与 MVP 静态验证

本阶段的目标是实现《MVP启动清单》中的核心需求，让新移民玩家能顺利“入境、在火车站区移动、消耗体力与金钱、并活过第一个 City Tick”。

1.  **项目初始化与环境配置**
    *   搭建 FastAPI 后端框架与 React 前端脚手架。
    *   配置 PostgreSQL 数据库与 Redis 缓存连接。
    *   建立前后端 WebSocket 通信机制。
2.  **核心时间与状态引擎 (Tick Engine)**
    *   实现世界时钟与个人时钟的换算逻辑。
    *   构建 City Tick 定时结算任务（属性扣除、状态更新）。
3.  **单窗口界面原型 (UI Framework)**
    *   根据《UI界面方案 (方案 B)》，实现深灰偏蓝背景的极简美学终端界面。
    *   开发顶部状态栏、中央叙事流（支持打字机效果）、底部指令输入框。
    *   实现内联手机系统组件的基础展示。
4.  **火车站隔离区场景与移动逻辑**
    *   定义 4 个基础 POI（入境、服务中心、便利店、收容所）。
    *   实现移动指令解析与时间/体能消耗计算。

**需要您提供的数据 (节点 2)：**
在开始第一阶段代码编写前，需要您提供以下 MVP 数据：
*   初始体能与饥饿上限。
*   基础消耗速率（每 GT 小时增加的饥饿值）。
*   火车站区内移动的平均时间与体能消耗。
*   便利店第一件商品的名称、价格及恢复数值。

### 第二阶段：核心系统扩展与持久化

在 MVP 验证通过后，本阶段将引入数据库持久化，并扩展经济与身份系统。

1.  **用户注册与状态持久化**
    *   实现玩家注册登录流程（统一难民制起点）。
    *   将玩家状态（属性、坐标、资产）保存至 PostgreSQL。
2.  **经济系统基础模块**
    *   实现现金与银行存款的双资产结构。
    *   开发基础的交易逻辑（购买商品、扣除手续费）。
3.  **动态公式解析引擎**
    *   构建能够解析您提供的文本公式的引擎，用于计算各类属性变化。
4.  **环境交互与文本反馈**
    *   实现基础的环境调查指令，展示您编写的区域描述文本。

**需要您审查的内容 (节点 3)：**
*   第二阶段的系统架构设计。
*   数据库表结构设计草案。

### 第三阶段：社会化模块与异步交互

本阶段将引入游戏的多人异步交互特性，丰富城市的社会生态。

1.  **职业与社会阶层系统**
    *   实现基于标签 (Tag) 的职业身份获取机制。
    *   开发账期机制（固定发薪日与账单日自动结算）。
2.  **实物交互与痕迹系统**
    *   实现便条系统与物品传递逻辑。
    *   开发 ASCII 环境改造的持久化存储。
3.  **NPC 行为框架**
    *   接入简单的行为树与对话分支解析器。
4.  **数字生存系统完善**
    *   完善手机 App 容器，实现异步简讯功能。

**需要您审查的内容 (节点 4)：**
*   社会化模块的交互逻辑演示。
*   NPC 数据结构的录入格式。

### 第四阶段：全服同城部署与运维体系

本阶段的目标是将游戏推向生产环境，支持多玩家同时在线。

1.  **分区服务器架构优化**
    *   引入乐观锁机制防冲突。
    *   实现空区块休眠逻辑以节省资源。
2.  **城市日报生成系统**
    *   开发数据抓取脚本，为 AI 生成日报提供数据源。
3.  **安全与防作弊机制**
    *   完善接口签名与操作频率限制。
4.  **生产环境部署**
    *   配置 CI/CD 流水线，完成正式环境的上线。
