# 机制改进与远期规划

本文档记录制作人在第一阶段中后期审查中提出的机制层面的改进方向和远期规划要点。这些内容不属于当前阶段的紧急修复，但需要在架构设计中提前预留接口。

---

## 一、游戏入口流程重构（远期）

### 当前状态

`mvp-src/client/src/pages/Home.tsx` 中的 `GamePhase` 流程为 `intro → characterCreation → transit → gameplay`。每次刷新页面都从 `intro`（边境站入境）开始，因为 `GameContext.tsx` 的 `initialState.phase` 硬编码为 `'intro'`。

### 制作人要求

角色注册与入境流程仅在账号首次创建时触发一次。后续登录应从主界面或上次下线位置继续游戏。

### 远期方案

接入后端后，`GamePhase` 流程将重构为：新玩家走 `login → intro → characterCreation → transit → gameplay`，老玩家走 `login → gameplay`（从数据库恢复坐标和状态）。`initialState` 不再硬编码，由后端 API 动态提供。

**实施时机**：后端基础设施（数据库 + 用户认证）就绪后。当前阶段不实施。

## 二、属性系统扩展预留

### 当前 PlayerStats

包含 `stamina`/`maxStamina`、`hunger`/`maxHunger`、`money`、`skills`（四维技能点）。

### 需要扩展的属性（待制作人定义数值后实装）

| 属性 | 说明 | 状态 |
| :--- | :--- | :--- |
| 生命值 `health`/`maxHealth` | 核心生存指标，归零触发死亡/昏迷 | 制作人已确认数值（见决策 #4） |
| 心理状态 | 长期孤独/压力可能影响行为效率 | 远期预留 |
| 信誉值 | 影响 NPC 态度、租房、求职 | 远期预留 |
| 犯罪记录 | 影响区域准入、NPC 互动 | 远期预留 |

工程端将在 `PlayerStats` 接口设计上保持可扩展性。

## 三、场景系统对齐（待确认）

### 当前场景

`GameContext.tsx` 中定义的场景为"世界公寓"体系：`apartment_lobby`（公寓大厅）→ `apartment_room`（房间，含厨房）→ `street`（公寓外街道）→ `convenience_store`（便利店）。

### MVP 启动清单要求的场景

原始 MVP 设计要求"火车站隔离区"，包含入境、服务中心、便利店、收容所 4 个 POI。

### 需要制作人确认

当前的"世界公寓"场景体系是否作为 MVP 的正式场景使用？还是需要按照启动清单重构为"火车站隔离区"？
