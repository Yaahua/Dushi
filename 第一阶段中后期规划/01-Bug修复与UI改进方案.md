## ~~Bug-01：叙事流文字应始终居中于屏幕视口~~ ✅ 已修复

**问题描述**：叙事流中的文字不断追加到底部，玩家视线需要持续跟踪页面底端。制作人要求最新一段文字始终出现在屏幕正中间，上一段文字向上推移。

**涉及文件**：`NarrativeStream.tsx`（`scrollIntoView` 滚动到底部）、`index.css`（`scene-enter` 淡入动画）。

**修复方向**：视口居中锚定 + 完全回滚查看（当前阶段），远期改为一天截断 + 日志系统。

**制作人决策**：见 `制作人决策记录.md` 决策 #1。

**修复提交**：`cad5af1` fix: 修复 Bug-01~04，实装生命值系统与饥饿联动机制

## ~~Bug-02：历史交互选项点击后应失效~~ ✅ 已修复

**问题描述**：叙事流中的超链接在被点击执行后依然可以反复点击，导致逻辑错乱。

**涉及文件**：`NarrativeStream.tsx`（`line.links` 渲染为 `<button>`）、`GameContext.tsx`（`ADD_NARRATIVE` 无失效标记）。

**修复方向**：`NarrativeLine` 增加 `expired` 字段，场景/状态变化时标记历史链接为失效，渲染时禁止点击但保留文字和下划线。

**制作人决策**：见 `制作人决策记录.md` 决策 #2。

**修复提交**：`cad5af1` fix: 修复 Bug-01~04，实装生命值系统与饥饿联动机制

## ~~Bug-03：缺少饥饿值与体能值的联动机制~~ ✅ 已修复

**问题描述**：体能值和饥饿值完全独立计算，体能没有自然恢复途径，饥饿值也不影响体能。

**涉及文件**：`GameContext.tsx`（`GT_TICK` 仅增加饥饿值；`COMPLETE_PROGRESS` 吃饭仅降低饥饿值；`GOTO_LOCATION` 扣除体能无恢复）。

**修复方向**：进食同时恢复体能，受饥饿阈值限制。

**制作人决策**：见 `制作人决策记录.md` 决策 #3。

**修复提交**：`cad5af1` fix: 修复 Bug-01~04，实装生命值系统与饥饿联动机制

---

## ~~Bug-04：缺少生命值（Health/HP）属性~~ ✅ 已修复

**问题描述**：MVP 要求实现死亡/昏迷判定逻辑，但 `PlayerStats` 不包含生命值属性。

**涉及文件**：`GameContext.tsx`（`PlayerStats` 接口）、`StatusBar.tsx`（仅显示体能、饥饿、金钱）。

**修复方向**：增加 `health`/`maxHealth` 属性，满饥饿后开始扣除生命值，归零触发昏迷。

**制作人决策**：见 `制作人决策记录.md` 决策 #4。

**修复提交**：`cad5af1` fix: 修复 Bug-01~04，实装生命值系统与饥饿联动机制
