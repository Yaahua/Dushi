# 《都市余烬》仓库文件分析笔记

## 项目概述
- **项目名称**：都市余烬 (Dushi)
- **类型**：纯文字、单窗口、异步多人的现代都市生活模拟器
- **核心概念**：全服玩家共享同一物理城市，作为新移民从零开始生存

## 文件清单与内容摘要

### 1. 未细化暂定大纲.md（总纲 v3.0/v3.1）
- 游戏定义、核心架构（8大系统）、技术路径、排除议题
- 8大核心系统：时间、出身注册、经济、生命死亡、身份职业、城市基建、叙事事件、交互界面
- 技术路径：静态验证(MVP) → 全服同城(分区服务器)
- 运算：乐观锁、事件队列快照、WebSocket长连接

### 2. 开发规划.md（模块化架构）
- 动态数据驱动架构
- 12个模块(A-L)：Tick引擎、UI框架、属性逻辑、地理物理、职业阶层、教育认证、经济金融、NPC智能、法律冲突、交通移动、数字通讯、环境痕迹
- 制作人定义内容，AI实现技术框架

### 3. 技术架构与交付清单.md
- **后端**：Python (FastAPI) / Node.js (Express)，微服务/模块化
- **数据库**：PostgreSQL
- **缓存/队列**：Redis
- **前端**：Web-based Terminal (React + WebSocket)
- 内容交付分5大类：城市规则数值、地理设施、社会职业政策、角色NPC社交、交互工具系统

### 4. UI界面方案.md（方案B：现代极简美学）
- 设计哲学：去除冗余、强调质感
- 配色：TrueColor灰阶（深灰蓝背景 #1A1B26）
- Unicode增强边框与图标
- 三区域布局：状态栏、叙事流、输入区
- 手机系统作为内联组件
- 打字机效果、状态闪烁、模糊地图

### 5. MVP启动清单.md
- MVP目标：新移民入境、火车站区移动、消耗体力金钱、活过第一个City Tick
- 三大核心模块：时间状态引擎、单窗口界面原型、火车站隔离区场景
- 待定数据：初始状态值、移动代价、便利店商品

## 技术栈总结
| 层级 | 技术选型 |
|------|---------|
| 后端 | Python (FastAPI) 或 Node.js (Express) |
| 数据库 | PostgreSQL |
| 缓存/队列 | Redis |
| 前端 | React + WebSocket (Web Terminal) |
| 部署 | 渐进式：MVP → 分区服务器 |
