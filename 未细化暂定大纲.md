# 《都市余烬》宏观设计文档 v4.0

## 一、游戏定义

纯文字、单窗口、异步多人的现代都市生活模拟器。全服玩家共享同一物理城市，作为新移民从零开始生存。

## 二、核心架构

### 1. 时间系统

双层时钟：世界时钟（集体节律）与个人时钟（个体流速）。离线暂停：下线时游戏时间冻结，仅保留后台消耗（饥饿/房租倒计时）。离散运算：City Tick 机制，分区计算，空区块休眠。

### 2. 出身与注册

统一难民制：所有玩家同一起点（火车站），无差别初始资金与装备。区域隔离：新移民受限于初始区域，需满足条件后进入主城。防作弊。

### 3. 经济系统

双资产结构（随身现金/银行存款），转账延迟到账。账期机制：固定发薪日与账单日，逾期有软性物理后果（停水/驱逐）。交易限制：现金上限与单笔限额，大额申报与延迟审核。

### 4. 生命与死亡

阶段式生命：自选年龄段（青年/中年/老年），决定可被杀与自然死亡概率。自然死亡：长期离线+角色年龄达标后触发失踪死亡。他杀机制：高门槛犯罪（需武器获取、侦查跟踪、三要素满足），非零成功率，严惩罚（现实封禁+游戏内标记）。继承制度：关系人公证继承，遗产税累进，无人继承则充公拍卖。

### 5. 身份与职业

流体身份：无职业选择界面，通过玩家自己选择做的事情获得标签（学生/雇员/店主）。标签维护。许可制度：高级职业需前置证书与考试，玩家可自建机构发放许可。

### 6. 城市基础设施

三层架构：底层（开发者控制的公立系统）、中层（可接管的经营性设施）、顶层（纯玩家自建生态）。玩家共建：土地竞拍、建筑建造、公司开设需跑现实流程（注册/资本/勘察）。物理限制：地块数量有限，交通耗时真实，物品占格与溯源。

### 7. 叙事与事件

三层事件池：环境基础事件、职业专属事件、跨时间连锁事件。触发机制：标签匹配（状态+地点+时间），非纯随机概率。异步交互：实物载体（便条/痕迹/物品传递），玩家可以游戏内碰面随后交换电话。城市日报：AI 抓取异常数据与玩家行为生成，分日报/周报/年报。

### 8. 交互界面

终端美学：纯文字、等宽字体、命令行输入、内联面板（无弹窗）。动作语法：颗粒化动作（微动作/复合动作），空间距离可视化。状态呈现：ASCII 符号表示环境、物品、关系历史。

## 三、技术栈选型

| 层级 | 技术选型 | 选择理由 |
| :--- | :--- | :--- |
| 后端框架 | Python (FastAPI) | 原生异步 + WebSocket 支持，适合 Tick 运算与实时推送 |
| 数据库 | PostgreSQL | 复杂 JSON 存储、强事务处理，适合双资产经济系统 |
| 缓存/队列 | Redis | 高速缓存 + Pub/Sub 机制，契合全服状态同步 |
| 前端框架 | React + WebSocket | 模块化终端 UI 组件，实时交互反馈 |
| 部署方案 | Docker + Nginx | 容器化便于环境隔离与扩展 |

部署路径：静态验证（MVP）→ 全服同城（分区服务器）。运算架构采用乐观锁防冲突、事件队列快照、WebSocket 长连接轻量维护。

## 四、技术模块清单

本项目采用**动态数据驱动 (Data-Driven)** 架构。制作人负责规划城市的所有子系统、政策、文本与数值；工程端负责构建通用的技术框架。以下模块在技术实现上地位平等，制作人可随时增加新模块。

| 模块 | 名称 | 关键技术点 |
| :--- | :--- | :--- |
| A | 核心时钟与数据调度 | 异步计算、全服状态同步、离线数据补算 |
| B | 通用界面渲染 | 三区域动态布局、颜色管理、内联组件 |
| C | 属性与状态逻辑 | 动态公式解析引擎 |
| D | 地理与物理环境 | 坐标系建立、信号衰减、区域准入校验 |
| E | 职业与社会阶层 | 标签系统、权限校验、薪资账期结算 |
| F | 教育与认证系统 | 考试题库加载、证书发放、社区课程注入 |
| G | 经济与金融政策 | 双资产管理、交易限额、税收与账单扣除 |
| H | NPC 行为与记忆 | 行为树框架、对话分支解析、关系持久化 |
| I | 法律、秩序与冲突 | 成功率计算引擎、惩罚状态机 |
| J | 交通与移动系统 | 移动代价计算、交通工具班次逻辑 |
| K | 数字生存与通讯 | 异步消息队列、App 容器化展示 |
| L | 环境痕迹与实物交互 | 便条系统、物品追踪、环境改造持久化 |
