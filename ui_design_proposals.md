# 《都市餘燼》UI 介面細化方案 (v1.0)

## 引言

《都市餘燼》的核心美學在於「純文字終端（CLI）」，這是一種利用限制來創造深度沈浸感的藝術。本文件旨在細化原有的「單窗口三區域」設計，提供三種不同風格的 UI 方案，並探討如何利用現代終端技術（如 ANSI 顏色、ASCII 框線、FIGlet 字體）來提升玩家的視覺體驗。

---

## 一、 UI 設計核心原則

1.  **資訊密度平衡**：文字遊戲容易讓玩家感到疲勞。我們需要透過空間（空白）、顏色區分與邊框來引導玩家視線。
2.  **無感切換**：手機、背包、地圖等功能應以「內聯展開（Inline Expand）」或「覆蓋層（Overlay）」形式出現，不破壞主敘事流的連貫性。
3.  **動態反饋**：雖然是文字，但透過打字機效果、閃爍的光標、進度條動畫，可以增加遊戲的「活生感」。

---

## 二、 三大 UI 風格方案

### 方案 A：經典終端黑客風 (Classic Terminal)
*   **視覺特徵**：綠/琥珀色單色調，高對比度，大量使用 `[ ]`、`< >`、`#` 等符號。
*   **適用場景**：強調「生存系統」的嚴肅感與「命令行互動」的硬核感。
*   **佈局細節**：
    *   **狀態欄**：使用簡單的 ASCII 分隔符。
    *   **側邊欄**：隱藏式，僅在輸入 `查看 狀態` 時以彈窗形式出現。
    *   **敘事流**：純文字，關鍵詞加粗或閃爍。

### 方案 B：現代極簡美學風 (Modern Minimalist)
*   **視覺特徵**：使用 TrueColor (24-bit) 顏色，柔和的灰色調背景，圓角框線（利用 Unicode 字符），FIGlet 大字體標題。
*   **適用場景**：強調「都市生活」的質感與「手機系統」的現代感。
*   **佈局細節**：
    *   **手機系統**：模擬現代手機介面，使用 `○` 代表 App 圖示，左右滑動感。
    *   **地圖**：利用不同深淺的區塊（Blocks）組成模糊的城市輪廓。
    *   **交互**：支持鼠標點擊關鍵詞觸發指令（利用 XTerm 鼠標協議）。

### 方案 C：賽博朋克霓虹風 (Cyberpunk Neon)
*   **視覺特徵**：青色、粉色與紫色的強烈對比，大量使用雙線邊框 (`║`, `═`)，背景帶有淡淡的 ASCII 雜訊。
*   **適用場景**：強調「黑市」、「犯罪」與「都市夜生活」的混亂美感。
*   **佈局細節**：
    *   **警告系統**：當玩家處於危險或被通緝時，全螢幕邊框變紅並閃爍。
    *   **資訊流**：訊息帶有隨機的「解碼中」動畫效果。

## 三、 核心組件細化設計

### 1. 狀態欄 (Status Bar)
不再只是簡單的一行字，而是分為三個邏輯區塊：
*   **左側 [環境]**：日期、天氣圖標（如 `☀`, `☁`, `☔`）、當前區域。
*   **中間 [核心]**：金錢（動態變色，赤字變紅）、生命/體力條（使用 `█` 代表）。
*   **右側 [通知]**：手機信號格、未讀訊息圖標 `✉`、當前標籤（如 `[新移民]`）。

### 2. 中央敘事流 (Narrative Stream)
*   **對話區分**：NPC 對話使用特定顏色，並帶有頭銜標籤。
    *   `[警衛]: "身份證拿出來。"` (警衛文字為黃色)
*   **系統提示**：使用區塊背景色區分，例如離線簡報使用深灰色背景。
*   **交互亮顯**：場景中可交互的物品（如「長椅」、「ATM」）在文本中以底線或高亮顯示。

### 3. 手機系統 (Mobile OS)
當玩家輸入 `打開 手機` 時，中央區域會彈出一個固定寬度的內聯窗口：
```text
┌─── 📱 城市移動終端 v4.0 ───┐
│ 10:42 AM   📶 5G   🔋 88%  │
├────────────────────────────┤
│ [🗺 地圖]   [🚗 叫車]       │
│ [📨 簡訊]   [📋 客軟體]     │
│ [☎ 撥號]   [🚨 緊急]       │
├────────────────────────────┤
│ 💡 提示: 深夜叫車請注意安全 │
└────────────────────────────┘
```

## 四、 技術實作建議

為了實現上述 UI，我建議在開發階段採用以下技術路徑：

1.  **組件化開發**：
    *   開發 `Box` 組件：處理所有帶框線的窗口，支持自動計算寬高。
    *   開發 `TextFlow` 組件：處理自動換行、打字機效果與滾動緩衝。
2.  **顏色管理**：
    *   定義一套「語義化顏色表」（例如：`Warning: Red`, `Success: Green`, `NPC_Talk: Cyan`），方便後續一鍵切換 UI 風格。
3.  **佈局引擎**：
    *   實作一個簡單的「Flexbox 模擬器」，用於在終端內分配區域權重，確保在不同尺寸的終端窗口下都能正確顯示。

## 五、 下一步建議

您可以在這三種風格中選擇一種作為**基調**，或者告訴我您對某個特定組件（如地圖、犯罪預備鏈介面）的具體視覺想像，我將為您繪製更詳細的 ASCII 原型圖。
