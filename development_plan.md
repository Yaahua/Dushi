# 《都市余烬》详细开发规划文件

## 引言

本文件旨在为《都市余烬》这款纯文字、单窗口、异步多人的现代都市生存模拟器，提供一份详细的开发规划。基于您提供的《都市余烬》完整设计共识 v2.3.1，并考虑到您将主导 NPC 文本与交互逻辑，而我（Manus）将负责技术实施与系统架构的分工模式，本规划将开发流程拆解为数个阶段，旨在确保开发的顺序性、稳定性与效率。

## 总体开发策略

我们将采用**最小可行性产品（MVP）**的迭代开发策略，优先构建游戏的核心机制与基础体验，然后逐步扩展功能与内容。这种方法有助于在早期验证核心玩法的可行性，并在每个阶段都能获得可测试的成果。每个阶段的完成都将作为下一个阶段的基础，确保系统的稳定性与可扩展性。

## 开发阶段划分与分工

以下是《都市余烬》的详细开发阶段划分，以及您与我之间的分工说明：

### 阶段一：核心系统基础搭建

**目标**：建立游戏运行的最基本框架，包括核心时间系统、基础界面与数据存储机制。

| 模块名称 | 您的主导内容 | 我的技术实施 | 预期成果 |
| :--- | :--- | :--- | :--- |
| **City Tick 系统** | 定义“世界时钟”与“个人时钟”的具体时间流逝比例，以及离线策略的触发条件。 | 实施双层时钟系统，确保在线与离线状态下的时间正确流逝。设计并实施异步运算框架。 | 核心时间系统稳定运行。 |
| **单窗口界面框架** | 提供界面元素的文字描述与排版要求，特别是 ASCII 艺术风格。 | 搭建基于文字的单窗口界面框架，实现中央叙事流、顶部状态栏与底部指令输入区。 | 游戏界面原型完成，符合纯文字终端美学。 |
| **数据库设计** | 规划所有需要持久化存储的数据类型。 | 设计并建立游戏的数据库结构，包括玩家、城市、物品、NPC 等核心表格。 | 游戏数据能被有效存储与读取。 |

### 阶段二：基础生存与交互

**目标**：让玩家能够创建角色，体验最基本的生存循环。

| 模块名称 | 您的主导内容 | 我的技术实施 | 预期成果 |
| :--- | :--- | :--- | :--- |
| **玩家角色与状态** | 定义角色创建点数分配规则、年龄影响、基础状态（饥饿、健康）数值。 | 实施角色创建系统与基础状态系统，处理数值变化并与时间系统联动。 | 玩家能创建角色，状态能正确显示。 |
| **基础移动与指令** | 提供基础移动指令与核心交互指令的文本。 | 实施指令解析器，开发基础的场景切换与位置更新逻辑。 | 玩家能通过文字指令进行移动。 |
| **经济系统基础** | 确定初始资金、账期、房租等数值。 | 实施双资产结构（现金/存款）与账期自动结算机制。 | 基础经济系统运行。 |
| **初始难民教学** | 编写难民服务中心教学文本与测试题。 | 实施入境流程、限制区域（火车站区）与强制教学流程。 | 新玩家能顺利完成入境。 |

| 模块名称 | 您的主导内容 | 我的技术实施 | 预期成果 |
| :--- | :--- | :--- | :--- |
| **手机功能实施** | 规划地图、简讯、叫车、客软件等文本内容与交互逻辑。 | 实施手机 UI 切换逻辑与各项功能（模糊定位、延迟简讯等）。 | 手机系统完整可用。 |
| **实物社交** | 编写便条系统文本、物品传递记忆逻辑、环境改造文本。 | 实施便条系统、物品传递追踪与环境痕迹持久化。 | 玩家能通过实物进行弱交互。 |
| **离线策略托管** | 定义离线策略文本与随机事件。 | 实施策略选择界面与离线进程模拟，生成“期间简报”。 | 离线策略与简报功能生效。 |

## 分工原则

*   **您**：作为制作人与内容创作者，专注于**游戏的灵魂**——所有文本内容、数值平衡决策、交互逻辑设计。
*   **我**：作为 AI 开发助手，专注于**游戏的骨架**——所有底层技术实施、系统架构、界面开发与性能优化。
