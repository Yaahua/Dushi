# 《都市余烬》UI 界面细化方案 (方案 B：现代极简美学风)

## 一、 设计哲学：现代极简美学 (Modern Minimalist)

方案 B 的核心是**“去除冗余，强调质感”**。它摒弃了传统终端界面的杂乱感，借鉴现代生产力工具（如 Notion, Slack）的视觉逻辑，旨在为玩家提供长时间阅读也不会感到疲劳的沉浸式生存体验。

### 1. 核心视觉原则
*   **空间感 (Whitespace)**：通过合理的行间距与区域留白，引导玩家视线，避免文字堆砌感。
*   **语义化颜色 (Semantic Colors)**：颜色不只是装饰，而是状态的直接反馈。
*   **Unicode 增强**：利用 Unicode 字符（如圆角边框、象形图标）提升界面的精致度。

---

## 二、 视觉规范 (Visual Specifications)

### 1. 配色方案 (Color Palette)
使用 TrueColor (24-bit) 灰阶层次，营造冷峻且现代的都市氛围。

| 用途 | 颜色代码 (Hex) | 视觉效果描述 |
| :--- | :--- | :--- |
| **背景 (Background)** | `#1A1B26` | 深灰偏蓝，像深夜的城市街道。 |
| **正文 (Text)** | `#C0CAF5` | 柔和的米白色，高易读性。 |
| **强调色 (Accent)** | `#7AA2F7` | 电子设备、手机、关键交互点。 |
| **警告 (Warning)** | `#F7768E` | 饥饿、流血、非法区域、通缉。 |
| **成功 (Success)** | `#9ECE6A` | 交易完成、获得技能、任务达成。 |

### 2. 边框与符号 (Borders & Symbols)
*   **主窗口边框**：使用圆角字符 `╭ ╮ ╰ ╯ ─ │`。
*   **进度条**：使用 `█` (已填充) 和 `░` (未填充)。
*   **状态图标**：
    *   体能：`⚡`
    *   饥饿：`🍲`
    *   金钱：`💰`
    *   信息：`✉`
    *   信号：`📶`

## 三、 核心组件细化设计

### 1. 顶部状态栏 (Status Bar)
状态栏采用左右对齐布局，实时反映核心生存指标。
```text
╭─── 🏙 都市余烬 v2.3 ─────────────────────────── 2026-04-04 ───╮
│ [👤 亚华]  [💰 ¥2,450]  [⚡ 85/100]  [🍲 40/100]  [📶 5G] │
├──────────────────────────────────────────────────────────────┤
```

### 2. 中央叙事流 (Narrative Stream)
*   **场景描述**：首行缩进，使用高亮度文字。
*   **NPC 对话**：`[姓名]: "对话内容"`，姓名颜色与对话内容区分。
*   **交互提示**：在文本中，可点击或可输入的关键词使用底线或特定颜色（如：`去 [便利店]`）。

### 3. 手机系统 (Mobile OS) - 方案 B 专属设计
手机不再是全屏菜单，而是作为叙事流中的“内联组件”弹出，带有半透明感（通过字符密度模拟）。
```text
│  ┌── 📱 智慧终端 ──────────────────────────────┐            │
│  │ 10:42 AM   📶 5G   🔋 88%                    │            │
│  ├──────────────────────────────────────────────┤            │
│  │  [🗺 地图]   [🚗 叫车]   [📨 简讯 (2)]        │            │
│  │  [📋 客软件] [☎ 拨号]   [🚨 紧急]            │            │
│  ├──────────────────────────────────────────────┤            │
│  │ 💡 提示: 您的账单将在 3 天后到期。           │            │
│  └──────────────────────────────────────────────┘            │
```

## 四、 交互反馈细节

### 1. 打字机效果 (Typewriter Effect)
文本根据叙事节奏动态流出，关键信息（如“你感觉非常饥饿”）流出速度稍慢，增加压迫感。

### 2. 状态闪烁 (State Pulsing)
当饥饿值低于 20% 时，顶部状态栏的 `[🍲 20/100]` 会以淡红色缓慢闪烁，提醒玩家寻找食物。

### 3. 模糊地图 (Blurred Map)
地图不显示精确像素，而是用不同密度的 ASCII 字符 (`.`, `:`, `*`, `#`, `@`) 组成模糊的区域密度图，模拟手机信号不佳时的加载感。

---

## 五、 技术实施建议 (给开发者的提示)

1.  **组件解耦**：将 UI 渲染分为 `Header`, `MainStream`, `InputArea` 三个独立类。
2.  **ANSI 封装**：建立一个颜色管理类，通过 `Color.accent("文字")` 自动转换为 ANSI 编码。
3.  **响应式布局**：计算终端窗口宽度，若宽度小于 80 字符，自动隐藏非核心状态图标。
