## 第一阶段

### 基础设施搭建与 MVP 静态验证

**目标**：实现 MVP 核心需求——新移民入境、火车站区移动、消耗体力金钱、活过第一个 City Tick。

**前置条件**：制作人确认技术栈选型、提供 MVP 数值数据（见 `mvp-checklist.md`）。

| 步骤 | 任务 | 交付物 | 审查节点 |
| :--- | :--- | :--- | :--- |
| 1.1 | 搭建 FastAPI 后端框架与 React 前端脚手架 | 可运行的空项目骨架 | 制作人确认项目结构 |
| 1.2 | 配置 PostgreSQL 数据库与 Redis 缓存连接 | 数据库连接测试通过 | 制作人确认 |
| 1.3 | 建立前后端 WebSocket 通信机制 | 双向通信演示 | 制作人确认 |
| 1.4 | 实现世界时钟与个人时钟换算逻辑 | Tick Engine 核心代码 | 制作人审查逻辑 |
| 1.5 | 构建 City Tick 定时结算（属性扣除、状态更新） | 结算流程演示 | 制作人测试并调整数值 |
| 1.6 | 实现单窗口三区域终端界面（状态条+叙事流+命令行）及内联卡片渲染 | UI 界面截图/演示 | 制作人审查视觉效果 |
| 1.7 | 实现命令行智能补全机制与超链接点击交互 | 交互范式演示 | 制作人测试输入体验 |
| 1.8 | 定义火车站区为漫游链节点，实现分层拓扑移动与消耗计算 | 火车站场景可交互 | 制作人测试完整生存循环 |

## 第二阶段

### 核心系统扩展与持久化

**目标**：引入数据库持久化，扩展经济与身份系统。

**前置条件**：第一阶段通过制作人验收、制作人提供经济系统数值。

| 步骤 | 任务 | 交付物 | 审查节点 |
| :--- | :--- | :--- | :--- |
| 2.1 | 实现玩家注册登录流程（统一难民制起点） | 注册登录功能 | 制作人审查流程 |
| 2.2 | 玩家状态持久化至 PostgreSQL | 数据库表结构文档 | 制作人审查表结构设计 |
| 2.3 | 实现现金与银行存款双资产结构 | 经济系统基础代码 | 制作人审查逻辑 |
| 2.4 | 开发基础交易逻辑（购买商品、扣除手续费） | 交易流程演示 | 制作人测试 |
| 2.5 | 构建动态公式解析引擎 | 引擎代码 + 使用文档 | 制作人确认公式格式 |
| 2.6 | 实现环境调查指令与区域描述文本展示 | 环境交互演示 | 制作人审查文本效果 |

## 第三阶段

### 社会化模块与异步交互

**目标**：引入多人异步交互特性，丰富城市社会生态。

**前置条件**：第二阶段通过制作人验收、制作人提供职业/NPC 相关规划。

| 步骤 | 任务 | 交付物 | 审查节点 |
| :--- | :--- | :--- | :--- |
| 3.1 | 实现标签 (Tag) 职业身份获取机制 | 标签系统代码 | 制作人审查标签逻辑 |
| 3.2 | 开发账期机制（发薪日与账单日自动结算） | 账期结算演示 | 制作人测试 |
| 3.3 | 实现便条系统与物品传递逻辑 | 实物交互演示 | 制作人审查 |
| 3.4 | 开发 ASCII 环境改造持久化存储 | 环境痕迹演示 | 制作人审查 |
| 3.5 | 接入行为树与对话分支解析器 | NPC 交互演示 | 制作人审查 NPC 行为 |
| 3.6 | 完善手机 App 容器与异步简讯功能 | 数字生存系统演示 | 制作人审查 |
