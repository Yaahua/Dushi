## MVP 开发目标

让新移民玩家能顺利"入境、在火车站区移动、消耗体力与金钱、并活过第一个 City Tick"。

## 三大核心基础模块

### 模块一：核心时间与状态引擎

City Tick 结算逻辑、属性扣除逻辑、死亡/昏迷判定逻辑。

### 模块二：单窗口界面原型

状态栏、叙事流、指令框解析器。参照 `ui-spec.md` 中方案 B 的视觉规范实现。

### 模块三：火车站隔离区场景

定义 4 个 POI：入境、服务中心、便利店、收容所。实现移动代价逻辑。

## 制作人待提供数据

以下数据全部由制作人定义，工程端不得擅自填写默认值。在制作人提供之前，相关功能以占位符形式预留。

### 初始状态值

| 数据项 | 状态 |
| :--- | :--- |
| 体能上限 | 待制作人提供 |
| 饥饿上限 | 待制作人提供 |
| 基础消耗速率（每 GT 小时饥饿增量） | 待制作人提供 |
| 离线期间消耗规则 | 待制作人提供 |

### 移动代价

| 数据项 | 状态 |
| :--- | :--- |
| 火车站区内平均时间消耗（GT 分钟） | 待制作人提供 |
| 每次移动平均体能消耗 | 待制作人提供 |

### 便利店第一件商品

| 数据项 | 状态 |
| :--- | :--- |
| 商品名称 | 待制作人提供 |
| 商品价格 | 待制作人提供 |
| 恢复值（饥饿/体能） | 待制作人提供 |

## 协作流程

1. 制作人提供上述数据
2. 工程端将数据转化为底层逻辑代码
3. 工程端展示第一个生存循环效果，供制作人测试与调整
